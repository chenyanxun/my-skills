# Novel to Comic：Cursor 适配设计

## 目标

在保留 `novel-to-comic/` 当前目录的前提下，将该 Skill 改造成可由 Cursor 准确执行、可验证且易维护的项目内 Skill。不得向用户目录或 `.cursor/skills/` 复制文件。

## 采用方案

采用“Cursor 原生优先”的混合方案：

- Cursor 负责读取输入、分析章节、确认场景、构造提示词并调用 `GenerateImage`。
- Python 脚本负责目录创建、文件名净化、角色资料持久化和 URL 提取兜底。
- URL 内容优先通过 `WebFetch` 获取；无法获取时才执行 Python 脚本。

## 文件设计

- `SKILL.md`
  - 明确触发条件和 Cursor 工具边界。
  - 使用 Cursor 支持的图片比例。
  - 定义用户确认、批量生成、角色一致性和失败恢复流程。
- `references/prompting.md`
  - 提供适用于 `GenerateImage` 的提示词结构。
  - 说明如何使用参考图片保持角色一致性。
- `references/examples.md`
  - 覆盖 URL、纯文本、连续章节和失败恢复示例。
- `scripts/novel_to_comic.py`
  - 保留现有公开函数。
  - 增加可诊断错误、数据校验和稳定的命令行退出状态。
- `tests/test_novel_to_comic.py`
  - 覆盖路径净化、场景文件名、角色数据读写、损坏 JSON 和失败提取。
- `agents/openai.yaml`
  - 删除。它不是 Cursor Agent Skill 的必要元数据，且会造成运行时归属混淆。

## 工作流

1. 判断输入为 URL、文件路径或直接文本。
2. 获取正文；失败时说明原因并请求用户提供文本。
3. 分析小说名、章节、角色和 4 至 8 个关键场景。
4. 在生成图片前让用户确认场景与风格。
5. 将风格映射到 Cursor 支持的比例：`3:4`、`4:3`、`16:9` 或 `9:16`。
6. 创建输出目录并更新 `characters.json`。
7. 按场景调用 `GenerateImage`；从第二张开始传入适用的角色参考图。
8. 报告成功、失败和保存路径；单张失败不隐藏，允许重试。

## 错误处理

- 网页提取失败：返回明确失败原因，避免静默捕获所有异常。
- 角色数据损坏：抛出带文件路径的可读错误，不覆盖原文件。
- 非法名称：净化 Windows 不允许的字符，并处理保留名称、尾随点和空格。
- 图片生成失败：保留已完成图片，列出失败场景，不重复生成成功项。

## 验证

- 先编写失败测试，再修改脚本使测试通过。
- 检查 `SKILL.md` YAML、触发描述、文件引用和行数。
- 搜索并确认不再使用 `image_gen`、固定像素尺寸或不存在的 Cursor 工具。
- 运行 Python 单元测试和脚本命令行冒烟测试。

## 非目标

- 不实现完整网页爬虫框架。
- 不自动绕过登录、验证码或反爬限制。
- 不向用户级 Skill 目录安装文件。
- 不引入图像后处理或多格漫画拼接。
